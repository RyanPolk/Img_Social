Introduction

The project that we chose to do is an online image-sharing website that implements social aspects for a more engaging experience.

The problems that we are addressing are information management and data processing regarding the management of users, images, and interactions.

Problem solving and algorithms

Our project's primary problem revolves around efficiently uploading images and maintaining quick queries in our database. This is critical to the project to maintain user retention.

However, there are other minor problems our project will encounter such as:

  1. A way to upload images to our database in a manner that provides lossless compression. Here is a way to upload images to our database with a special type of encoding.
  2. A method to host our database that contains users and their respective comments and images. There are plenty of hosting sites that we could use to solve this problem. Here is a list of some solutions.
  3. An eye-catching and minimalistic design that will hold a user's attention. This is our inspiration for our design.
  4. To use atomic operations within our database to synchronize likes and comments made by users. This should not be a significant problem for our project.

  1. We plan to use the 3-Tier Web Application Model. We will have a browser-based client that interacts with our web server and web pages. The web server will then communicate with our back-end database.
